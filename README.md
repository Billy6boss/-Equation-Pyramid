# 方程式金字塔 Equation Pyramid
每回合會有1個目標數字，
以及10個包含運算符號與數字的格子，
例如：+5、×7、÷2等等……
參賽者要用3個格子組出算式來達到目標數字，
答對加1分，答錯扣1分。

## 遊戲核心規則與玩法 
### 目標： 
1. 在底層的數字格子中，挑選三個數字，依序組成算式
2. 目標需等於目標數字。
3. 計算順序： 採用先乘除、後加減的邏輯順序。
4. 格式： 選出的三個數字/符號，第一個數字/符號通常僅作為數字進行初步計算，例如若格子是 (+5,×7,÷2) 等組合。限制： 每題限時 3 分鐘，需在限定時間內找出正確答案。
5. 首位符號失效：你必須選擇 3 個方塊來組成等式。第一個方塊的運算符號不參與計算，系統只會提取該方塊上的數字。
6. 同一個答案不可以重複使用：在同一回合中，格子可以重複使用，但組成的算式必須是獨一無二的。


## 範例說明
範例說明：
如果你依序選擇 (-7)、(+5)、(×2)。
計算過程會是：7 + 5 × 2。
結果為：17（符合先乘除後加減原則）。


## 技術說明
- 使用平台: 網頁遊戲
- 開發語言: JavaScript, HTML, CSS 分開三個檔案
- 畫面呈現: 六邊形組成的金字塔形狀，每個六邊形內顯示一個數字或運算符號與上方有一個固定的英文字母，右邊會有一個目標數字顯示，下方會呈現當使用者選擇三個數字之後所形成的公式，最上面會有類似7段顯示器的倒數計時器，計時器旁邊有個開始按鈕，最左邊會有個可收縮的計分表，可以新增,顯示,刪除隊伍，讓每個小組加減分。
- 遊戲邏輯: 畫面一開始都沒不會顯示任何數字，只有字母，直到按下開始按鈕之後才會顯示符號數字與目標數字，使用者點擊六邊形選擇數字與運算符號，系統根據選擇的順序計算與顯示結果並與目標數字比較，正確則撥放正確音效(./sound/correct.mp3)，錯誤則撥放錯誤音效(./sound/incorrect.mp3)
- 目標數字生成: 先計算過所有可能的答案，再從中選擇出現頻率較高的數字作為目標數字。
